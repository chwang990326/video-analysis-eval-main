# 视频分析课题评测方案

### 一、 评测指标定义

评测指标分为定位精度、ID 稳定性、行为识别和身份识别四个维度：

**1. 定位精度**
衡量系统输出坐标与真实物理坐标的偏差。设系统输出的人体检测框底边中心点坐标为 (X_pred, Y_pred)，真实物理坐标为 (X_gt, Y_gt)。

*   **单帧距离误差 (E)**：
    计算公式：E = 根号下 [ (X_pred - X_gt)的平方 + (Y_pred - Y_gt)的平方 ]
*   **P50 误差（中位误差）**：将统计时间段内所有帧的误差 E 按从小到大排序，取第 50% 位置的误差值。
*   **P90 误差**：将统计时间段内所有帧的误差 E 按从小到大排序，取第 90% 位置的误差值。
*   **Max 误差**：统计时间段内所有帧的误差 E 的最大值。

**2. ID 稳定性**
衡量系统在连续视频流中对同一目标的跟踪维持能力（无论是否知道该目标是谁，只需保持 ID 编号不变）。
*   **轨迹跳变次数**：计算相邻两帧之间目标移动的速度 v。若 v 大于设定的速度阈值（如 5m/s），则记为 1 次跳变。
    计算公式：速度 v = 根号下 [ (当前帧X坐标 - 上一帧X坐标)的平方 + (当前帧Y坐标 - 上一帧Y坐标)的平方 ] / 两帧之间的时间差
*   **串号次数**：在设定的校验点，系统当前输出的当前ID与初始绑定的初始ID不一致的次数。
*   **误合并次数**：系统输出的检测框数量小于真实人数，且单个检测框包含多人的次数。
*   **ID 断裂次数**：目标未离开画面，但原有 ID 消失并被分配新 ID 的次数。

**3. 行为识别**
衡量系统对异常动作的检测能力。设 TP 为正确报警的事件数，FN 为漏报的事件数，FP 为误报的事件数。

*   **检出率 (Recall)**：
    计算公式：检出率 = [ TP / (TP + FN) ] * 100%
*   **报警延迟 (Latency)**：
    计算公式：报警延迟 = 系统首次输出报警的时间戳 - 真实动作开始的时间戳
*   **归属准确率**：
    计算公式：归属准确率 = (绑定到正确 ID 的报警次数 / TP) * 100%
*   **误报频次 (False Alarm)**：在非动作发生时间段内，或执行正常干扰动作时，系统输出报警的总次数。

**4. 身份识别（人脸/ReID）**
衡量系统将画面中的目标与底库人员真实身份正确匹配的能力。
*   **身份识别准确率**：针对已注册入库的人员，系统输出的身份结果与真实身份一致的比例。
    计算公式：身份识别准确率 = 识别正确的总次数 / 已注册人员出现的总校验次数 * 100%
*   **陌生人拒识率**：针对未入库的陌生人，系统正确将其标记为“未知”或“未注册”的比例。
    计算公式：陌生人拒识率 = 正确标记为未知的次数 / 陌生人出现的总校验次数 * 100%
*   **身份确认延迟**：目标进入画面到系统首次输出正确身份标签的时间差。
    计算公式：身份确认延迟 = 系统首次输出正确身份的时间戳 - 目标进入画面的时间戳

### 二、 评测方法说明

评测方法通过设定固定点位和时间段来验证系统逻辑，不依赖全过程的逐帧坐标比对：

1.  **定位精度评测**：不在人员走动过程中计算误差。仅在人员站在已知坐标的固定点位上静止时，提取系统输出的坐标与真实坐标计算误差。
2.  **ID 稳定性评测**：
    *   **速度校验**：通过计算相邻帧的坐标位移速度，超过合理阈值即判定为轨迹跳变。
    *   **点位校验**：要求人员在特定时间经过特定的已知点位。检查该时刻该点位上的 ID 是否还是初始分配的 ID，如果不一致，则判定发生串号。
3.  **行为识别评测**：通过比对时间窗口。检查在标注的动作起止时间内，系统是否针对对应的目标 ID 输出了正确的动作标签。
4.  **身份识别评测**：在“初始站位”和“中间打卡”的时间窗口内，提取系统为该目标 ID 赋予的“身份标签（如姓名/工号）”，与标注的真实身份进行字符串比对。

### 三、 数据集准备与场景录制流程

数据集准备需覆盖不同的物理空间、人员数量、行为特征及身份注册状态，具体分为以下四个步骤：

**1. 场地测量与标定**
在测试场地地面设置 9 个固定点位（P1~P9，呈九宫格分布）。使用激光测距仪测量这些点位在世界坐标系下的真实 (X, Y) 坐标，并记录为系统配置文件。

**2. 评测维度设计**
录制素材需严格覆盖以下变量维度：
*   **人数维度**：单人场景、多人场景（3-5 人）。
*   **距离维度**：近场（距镜头 2-5 米）、中场（5-10 米）、远场（10 米以上）。
*   **朝向维度**：正面（利于人脸识别）、背面/侧面（考验 ReID 能力）。
*   **遮挡维度**：无遮挡、人员相互遮挡、被环境物体短暂遮挡。
*   **身份注册维度**：底库已注册人员、底库未注册人员（陌生人）。

**3. 录制场景与剧本设计**
在系统底库中提前录入部分测试人员的人脸和特征信息。按设计的剧本录制视频：
*   **场景一：单人静态与朝向测试（主测定位与多角度身份识别）**
    单人依次走到 P1 至 P9 点位。在每个点位上，分别面向镜头的正前方、正后方、左侧、右侧各静止站立 3 秒。
*   **场景二：单人动态与行为测试（主测行为检出与延迟）**
    单人从 P1 出发，按 S 型路线走过所有点位。在规定的点位执行异常动作（如摔倒、抽烟），动作需持续 3-5 秒。
*   **场景三：多人交叉走位测试（主测 ID 串号与误合并）**
    3 名人员分别站在 P1、P2、P3 静止 3 秒（初始绑定）。随后 3 人同时向场地另一侧走动，发生多次路线交叉和身体重叠。剧本要求某人在特定时间必须经过 P5 点位并稍微停顿。
*   **场景四：干扰动作测试（主测误报率）**
    人员在场地内执行易与异常行为混淆的正常动作（如蹲下系鞋带、喝水）。
*   **场景五：陌生人混入测试（主测拒识率与 ReID 稳定性）**
    1 名已注册人员与 1 名未注册人员（陌生人）同时进入画面，进行交叉走位。要求陌生人在特定时间经过特定打卡点。

**4. 数据标注规范**
标注人员观看视频，记录关键事件的时间段、对应的人员真实身份、注册状态、位置或动作。标注格式示例：
*   `[00:00-00:03]` 真实身份：张三（已注册），位置 P1，朝向：正面（初始站位）
*   `[00:00-00:03]` 真实身份：未知（未注册），位置 P3，朝向：正面（初始站位）
*   `[00:15-00:16]` 真实身份：张三（已注册），位置 P5（经过中间点打卡）
*   `[00:25-00:30]` 真实身份：张三（已注册），动作：摔倒

### 四、 自动化评测执行流程

基于上述数据集，通过脚本自动执行以下流程：

1.  **数据采集**
    脚本按设定帧率抽取视频帧，调用视频分析系统 API，记录每一帧返回的 `Target_ID`、`X`、`Y`、`Action`、`Recognized_Name`（识别出的身份）及时间戳。
2.  **ID 映射绑定**
    读取视频前 3 秒的数据，将系统输出的坐标与初始站位的真实坐标进行距离匹配。将系统生成的 `Target_ID` 与真实人员进行物理绑定。
3.  **指标计算**
    脚本遍历采集到的数据执行计算：
    *   **计算误差**：在初始站位和打卡点时间段内，提取对应 ID 的坐标与真实点位计算 P50/P90/Max 误差。
    *   **判断串号与跳变**：在中间打卡点时间段内，检查对应位置的 ID 是否与初始绑定一致，不一致则记录串号；全局扫描坐标跳变速度，记录异常跳变次数。
    *   **判断行为**：在标注的动作时间段内，检查对应的 ID 是否输出了正确的动作标签。
    *   **判断身份**：在初始站位和打卡点时间段内，比对系统输出的 `Recognized_Name` 与标注的真实身份。若标注为已注册，比对名称是否一致；若标注为未注册，检查系统是否输出“未知”。统计准确率和拒识率。
4.  **结果输出**
    *   生成包含定位精度、ID 稳定性、行为识别率、身份识别率等各项量化指标的汇总表格。
    *   针对发生跳变、串号、漏检、误报或身份识别错误的时间段，自动截取前后几秒的视频片段，并将系统输出的检测框、ID 和身份名称渲染在画面上，作为排查问题的素材。
