# 视频分析系统自动化评测操作执行手册

## 第一阶段：场地与系统准备

**步骤 1：物理点位标定**
1. 在摄像机覆盖的测试场地地面，使用胶带贴出 9 个固定点位，标记为 P1 至 P9（建议呈九宫格分布，覆盖近、中、远场）。
2. 设定场地的一个角落为坐标原点 (0,0)。
3. 使用激光测距仪，测量 P1 至 P9 每个点位中心到原点的真实物理坐标 (X, Y)，单位统一为厘米 (cm)。
4. 将点位坐标记录为 `location_config.json` 文件。
   ```json
   {
     "P1": {"x": 100, "y": 200},
     "P2": {"x": 300, "y": 200},
     "P9": {"x": 500, "y": 600}
   }
   ```

**步骤 2：底库人员注册**
1. 挑选 3 名测试人员（如张三、李四、王五），采集他们的人脸和全身照。
2. 将这 3 人的信息录入被测视频分析系统的“已知人员底库”中。
3. 安排 1 名**未录入底库**的人员作为“陌生人”（如赵六）参与后续测试。

---

## 第二阶段：视频录制执行

请摄影师开启录像，测试人员严格按照以下 4 个剧本进行表演。**核心规则：每次录制开始的前 3 秒，所有画面内的人员必须站在指定的点位上保持绝对静止（用于系统初始化绑定 ID）。**

**剧本 1：单人静态与身份识别测试**
*   **执行动作**：张三走到 P1 点位，面向镜头站立 3 秒；然后转身背向镜头站立 3 秒；接着走向 P2 点位，面向镜头站立 3 秒。依次走完 P1 到 P9。
*   **评测目的**：测试系统在不同距离、不同朝向下的定位精度和人脸/ReID 识别准确率。

**剧本 2：单人动态与行为测试**
*   **执行动作**：李四站在 P1 静止 3 秒。随后开始在场地内随意走动。走到 P5 点位时，执行“摔倒”动作，倒地保持 5 秒后起身。走到 P8 点位时，执行“抽烟”动作 5 秒。
*   **评测目的**：测试异常行为的检出率和报警延迟。

**剧本 3：多人交叉与轨迹稳定性测试**
*   **执行动作**：张三站 P1，李四站 P3，陌生人赵六站 P7。三人同时静止 3 秒。
*   **执行动作**：三人开始向场地中央走动，故意发生路线交叉、身体遮挡。
*   **执行动作**：在第 20 秒左右，张三必须经过 P5 点位并稍微停顿 1 秒（打卡校验点）。随后三人走出画面。
*   **评测目的**：测试 ID 串号、误合并，以及陌生人拒识率。

**剧本 4：干扰动作测试**
*   **执行动作**：王五站在 P2 静止 3 秒。随后在场地内走动，执行“蹲下系鞋带”、“弯腰捡笔”、“喝水”等日常动作。
*   **评测目的**：测试系统是否会将正常动作误报为异常行为（误报率）。

## 第三阶段：极简数据标注

无需使用画框工具，标注人员只需使用 Excel 或文本编辑器，边看视频边记录关键事件的时间轴。

**步骤 1：新建标注表格（Excel 或 CSV）**
创建包含以下列的表格：`视频名称`、`开始时间`、`结束时间`、`事件类型`、`真实姓名`、`注册状态`、`发生位置`、`动作标签`。

**步骤 2：按规则填写时间段**
根据录制的视频，逐行填写记录。以下为标准填写示例：

| 视频名称  | 开始时间 | 结束时间 | 事件类型 | 真实姓名 | 注册状态 | 发生位置 | 动作标签 |
| :-------- | :------- | :------- | :------- | :------- | :------- | :------- | :------- |
| 剧本3.mp4 | 00:00.0  | 00:03.0  | 初始站位 | 张三     | 已注册   | P1       | 无       |
| 剧本3.mp4 | 00:00.0  | 00:03.0  | 初始站位 | 李四     | 已注册   | P3       | 无       |
| 剧本3.mp4 | 00:00.0  | 00:03.0  | 初始站位 | 赵六     | 未注册   | P7       | 无       |
| 剧本3.mp4 | 00:20.0  | 00:21.0  | 中间打卡 | 张三     | 已注册   | P5       | 无       |
| 剧本2.mp4 | 00:00.0  | 00:03.0  | 初始站位 | 李四     | 已注册   | P1       | 无       |
| 剧本2.mp4 | 00:15.0  | 00:20.0  | 异常行为 | 李四     | 已注册   | P5       | 摔倒     |
| 剧本4.mp4 | 00:10.0  | 00:15.0  | 干扰动作 | 王五     | 已注册   | P2       | 系鞋带   |

*注：将该表格导出为 `annotations.csv`。*

## 第四阶段：运行自动化评测脚本

**步骤 1：准备目录结构**
在评测服务器或电脑上，按以下结构存放文件：
```text
/eval_workspace/
├── videos/                  # 存放录制好的 mp4 视频
├── config/
│   └── location_config.json # 物理点位坐标配置文件
├── annotations.csv          # 刚才填写的标注表格
└── run_eval.py              # 自动化评测执行脚本
```

**步骤 2：执行评测命令**
打开终端或命令行，输入以下命令启动全自动评测（假设被测系统的 API 地址为 `http://192.168.1.100:8080/analyze`）：

```bash
python run_eval.py --video_dir ./videos --anno_file annotations.csv --config ./config/location_config.json --api_url http://192.168.1.100:8080/analyze
```

**步骤 3：等待脚本自动处理**
脚本将自动完成以下工作，无需人工干预：

1. 逐帧读取视频并发送给被测系统 API。
2. 提取 `00:00-00:03` 的结果，将系统生成的 `Target_ID` 与坐标点上的真实姓名自动绑定。
3. 校验 `中间打卡` 时间点的 ID 是否发生串号，身份识别结果是否正确。
4. 校验 `异常行为` 时间段内是否触发了正确的报警，并计算延迟。
5. 全局扫描坐标移动速度，抓取轨迹跳变。

---

## 第五阶段：查看评测结果与复盘

脚本运行结束后，会在当前目录下自动生成一个 `output_results/` 文件夹。请查收以下两项核心交付物：

**1. 查阅量化指标报告 (`eval_report.xlsx`)**
打开该表格，直接查看系统的各项能力得分：
*   **定位精度**：P50 误差 (cm)、P90 误差 (cm)。
*   **ID 稳定性**：轨迹跳变总次数、ID 串号总次数、误合并次数。
*   **身份识别**：已注册人员识别准确率 (%)、陌生人拒识率 (%)。
*   **行为识别**：各动作检出率 (%)、平均报警延迟 (秒)、归属准确率 (%)、误报总次数。

**2. 审查错误切片视频 (`bad_cases/` 文件夹)**
进入该文件夹，您会看到脚本自动截取的错误瞬间短视频，例如：
*   `串号错误_剧本3_00m20s.mp4`
*   `漏报_摔倒_剧本2_00m15s.mp4`
*   `身份识别错误_把张三识别为李四_剧本1_00m10s.mp4`

这些视频画面上已经自动叠加了系统输出的检测框、ID 编号和识别名称。**请直接将此文件夹打包发送给算法研发团队，作为修复 Bug 的精准素材。**
